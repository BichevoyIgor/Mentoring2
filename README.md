### Задачи для практики с использованием Spring Data и Spring Web MVC:

#### Работа с Spring Data:

1. Создание простого репозитория:
    - Создайте проект Spring Boot с зависимостью Spring Data JPA.
    - Определите сущность `Product` с полями `id`, `name`, и `price`.
    - Создайте интерфейс репозитория `ProductRepository`, наследующий от `JpaRepository`.

2. Создание методов поиска в репозитории:
    - Добавьте в `ProductRepository` метод поиска продукта по `name`.
    - Реализуйте тесты для проверки работы этого метода.

3. Конфигурирование базы данных:
    - Настройте проект для использования встраиваемой базы данных H2.
    - Создайте скрипт инициализации базы данных для заполнения таблицы `Product` начальными данными.

4. Пагинация и сортировка:
    - Добавьте к репозиторию метод для получения страницы продуктов, отсортированных по `price`.
    - Настройте тест, проверяющий получение первой страницы продуктов с размером 5.

5. Работа с отношениями в Spring Data:
    - Определите сущности `Category` и `Product`, свяжите их через отношение `ManyToOne`.
    - Реализуйте репозиторий для `Category`, добавьте метод получения всех продуктов по категории.

#### Интеграция с Spring Web MVC:

6. Создание RestController для Product:
    - Создайте контроллер `ProductController`, который будет обрабатывать REST-запросы для сущности `Product` (
      CRUD-операции).

7. Валидация входных данных:
    - Добавьте валидацию для контроллера `ProductController` с аннотациями, такими как `@Valid` и `@NotNull`.

8. Работа с DTO и преобразованием:
    - Реализуйте DTO для сущности `Product` для обмена данными через API.
    - Настройте преобразование сущности в DTO и обратно в контроллере.

9. Использование аннотации `@RepositoryRestResource`:
    - Примените `@RepositoryRestResource` к репозиторию, чтобы автоматически создать RESTful интерфейс для
      сущности `Product`.

10. Обработка ошибок и исключений в контроллере:
    - Напишите глобальный обработчик исключений с помощью `@ControllerAdvice`, чтобы вернуть понятные сообщения об
      ошибках пользователю.

#### Расширенные возможности:

11. Использование Spring Data для работы с NoSQL:
    - Подключите зависимость Spring Data MongoDB.
    - Создайте сущность `Customer`, реализуйте репозиторий для нее, и протестируйте CRUD-операции с MongoDB.

12. Профилирование и конфигурация приложения:
    - Настройте разные профили для работы с различными базами данных (например, H2 для разработки и PostgreSQL для
      продакшена).

13. Тестирование контроллеров и репозиториев:
    - Напишите тесты для REST-контроллеров с помощью `@WebMvcTest`.
    - Используйте `@DataJpaTest` для тестирования JPA-репозиториев.

14. Реализация безопасных транзакций:
    - Настройте метод, обвешанный `@Transactional`, для создания заказа, включающего несколько продуктов.
    - Реализуйте логику отката в случае ошибок.

15. Интеграция Spring Security с Web MVC и Data:
    - Добавьте в проект Spring Security.
    - Настройте простую аутентификацию пользователей и авторизацию для доступа к данным через REST-контроллеры.
